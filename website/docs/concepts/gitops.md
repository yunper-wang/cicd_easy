# GitOps 核心概念

GitOps 是一种使用 Git 作为单一可信源的持续交付方法。

## 四大原则

1. **声明式配置** — 系统的期望状态用声明式语言描述
2. **版本控制** — 期望状态存储在 Git 中，拥有完整的变更历史
3. **自动部署** — 通过 Git 提交自动触发部署
4. **持续协调** — 控制器持续确保实际状态与期望状态一致

## 核心工具

| 工具 | 角色 | 说明 |
|------|------|------|
| Git | 配置存储 | 所有 K8s manifests、CI 配置存储在 Git 中 |
| Argo CD | 持续交付 | 监控 Git 变更，自动同步到集群 |
| GitLab CI | 持续集成 | 代码提交后自动构建、测试、推送镜像 |
